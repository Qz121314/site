# 后台信息架构

后台采用“全局管理 + 分区独立管理”结构。

```text
全局管理
├── 概览
├── 站点设置
├── 分区管理
└── 图片库

分区 A
├── 分区设置
├── 分类筛选
├── 分类管理
├── 产品管理
├── Hero 广告
└── 转化池

分区 B
├── 分区设置
├── 分类筛选
├── 分类管理
├── 产品管理
├── Hero 广告
└── 转化池
```

规则：

- 分区列表从 D1 动态读取。
- 新增、改名、排序或删除分区后，刷新后台即可更新左侧菜单。
- 分类筛选、分类、产品、广告池和转化池不是全局一级菜单。
- 每个业务查询和写入都必须固定使用当前 `channel_id`。
- 不允许产品绑定其他分区的分类或转化组。
- 不允许分区绑定其他分区的 Hero 广告池。
- 图片库属于全站共享 R2 资源，因此保留为全局菜单。
- 后台 URL 使用稳定分区 ID：`/admin/channels/{channelId}/{module}`。
