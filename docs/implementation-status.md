## 后台信息架构

后台按实际运营任务组织，而不是按数据库表逐个提供 CRUD 页面：

- 全局管理：概览、站点设置、分区管理、图片库
- 分区内容：产品信息录入、产品列表管理、分类筛选、Hero 广告、转化池
- 分区管理只负责名称、Slug、图标、排序和发布状态等低频配置
- 分区子菜单只保留日常高频内容运营入口
- 不提供独立“分类管理”菜单；分类目录由产品录入自动生成
- 图片库不承担素材上传，只负责 R2 审计、引用检查和清理

## 数据原则

- 产品是后台核心对象，分类目录由产品数据驱动生成。
- 产品只绑定分类；筛选项绑定分类目录，而不是直接绑定产品。
- 图片上传跟随具体业务对象，图片库只负责 R2 审计和清理。
- 项目自身不记录公开访问、点击或转化数据。
- GA4 和 Meta Pixel 仅在后台配置后注入前台。
- 公开查询只读取 `published` / `enabled` 数据。
- 图片由浏览器直接从 R2 自定义域名加载；后台鉴权预览仅作为未配置公开域名时的管理备用方案。
- 删除产品、分类绑定或图库关系不会立即删除 R2 对象；未引用图片统一由图片库清理。
- 公开页面不提供全站 noindex 开关；后台、管理 API 和 CTA 跳转端点继续禁止搜索引擎抓取。

## 运营配置流程

1. 在站点设置中配置网站名称、描述、Logo、Favicon 和 R2 公开域名。
2. 在分区管理中创建分区并完成低频结构配置。
3. 提前创建需要在前台展示的分类筛选项。
4. 在产品信息录入页面填写产品、直接上传图片并输入或选择分类。
5. 系统自动创建或复用分类目录。
6. 在分类筛选页面将生成的分类目录绑定到一个或多个筛选项。
7. 配置转化组和 Hero 广告，发布分区及产品。
8. 在图片库中定期检查和清理 R2 未引用图片。

运营数据保存在 D1/R2，保存后刷新前台即可生效，不需要重新部署代码。
