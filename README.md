# Cloudflare Personal Catalog Template

一个基于 **Astro SSR + Cloudflare Workers + D1 + R2** 的通用图片目录与转化站模板。

项目定位不是电商商城，而是一个高性能、可重复部署的商业目录模板，用于产品推荐、服务目录、本地商家目录、资源导航和联盟转化站。

核心路径：

```text
图片吸引用户
    ↓
浏览分区 / 分类 / 产品
    ↓
进入产品详情页
    ↓
阅读内容
    ↓
点击 CTA 完成转化
```

项目不提供：价格、库存、SKU、购物车、订单、支付、用户账户、收藏、评分、评论。

# 3. 冻结设计原则

以下方向已经确定，不再重新设计：

## 数据模型

```text
站点
 └── 分区 (1-4)
      ├── 筛选器
      ├── 分类
      ├── 产品
      ├── 广告组
      └── CTA 转化组
```

## 产品模型

统一模板兼容：

- 产品推荐
- 服务目录
- 文章内容
- 线下业务
- 联盟推广

不拆分多个模板。

## 页面原则

公共站点：英文

管理后台：中文

项目自身：

- 不统计访问数据
- 不统计点击数据
- 不统计收益数据

统计交给：

- GA4
- Meta Pixel

# 4. 公共站点结构

## 手机端（冻结）

```text
Header
广告区域
搜索
筛选器
双列分类
双列产品
底部分区切换
```

规则：

- 产品双列展示
- 分类双列展示
- 底部分区支持 1–4 个
- 按实际数量自动均分
- 不写死按钮宽度
- 产品详情当前页打开

---

## PC 端高信息密度工作区

PC 从 1100px 开始：

```text
顶部栏
Logo | 搜索 | Privacy | Disclaimer

左侧导航              主内容

分区 A                 分类区域 | 产品区域
  筛选 A                分类1 分类2 | 产品1 产品2
  筛选 B                分类3 分类4 | 产品3 产品4

分区 B
  筛选 A
```

规则：

- 左侧固定分区导航
- 展示所有有效筛选器
- 点击筛选器页面内定位
- 切换分区进入对应 URL
- 分类保持紧凑显示
- 产品区域固定列数
- 不因为产品少而拉伸卡片
- PC 产品详情新标签页打开

产品列：

| 屏幕 | 产品列 |
|---|---:|
| 1100-1359px | 3 |
| 1360-1759px | 4 |
| ≥1760px | 5 |

# 5. 图片规范

目录产品卡片统一：

```css
aspect-ratio: 4 / 5;
object-fit: cover;
object-position: center;
```

目标：

- 所有产品卡片高度一致
- 横图、竖图、方图统一展示
- 不影响详情页完整显示

推荐：

| 类型 | 尺寸 |
|-|-:|
| 产品封面 | 1200×1500 |
| 详情图片 | 1200×1500 |
| Logo | 600×180 |
| Favicon | 512×512 |

# 7. 性能目标

重点控制 Cloudflare 免费计划资源：

## Workers

- 减少无意义请求
- 减少重复计算
- 保持 SSR 高效

## D1

- 使用有效索引
- 避免大范围查询
- 只读取需要字段

## R2

- 原图与缩略图分离
- 列表只读取缩略图
- 延迟加载详情资源

## 前端

- 减少 JS
- 减少 CSS 重复覆盖
- 保持首屏速度
