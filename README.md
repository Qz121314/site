# Cloudflare Personal Catalog Template

一个基于 Astro SSR 与 Cloudflare Workers、D1、R2 构建的通用产品目录与转化站模板。公共站点面向美国市场，采用中性、清晰、图片优先的通用电商目录视觉；中文管理后台用于维护站点、分区、分类、产品、联盟广告组、转化池和图片资产。

本项目不是购物商城，不提供价格、库存、购物车、结账、订单、用户、收藏、评分或评论。核心目标是：

```text
图片吸引用户
  ↓
目录快速浏览
  ↓
进入产品详情
  ↓
点击 CTA
  ↓
完成链接、短信或复制转化
```

## 当前状态与目标规格

项目按空数据模板维护，可配置为产品、服务、资源、内容或本地商家目录站。

当前公共页面视觉、响应式目录、产品详情、搜索、CTA 转化池、D1/R2 数据链路、SEO 和部署验证已经建立。README 中的“联盟广告系统”章节是广告模块的目标要求，用于替换当前仅支持 Hero 图片加链接的旧广告模型；实现广告改造时必须以该章节为准，不能继续扩大旧 Hero 专用结构。

主要能力：

- 英文公共站点
- 中文管理后台
- 后台使用左侧固定菜单，不使用顶部导航菜单
- 多分区内容管理
- 可选分类与快捷筛选
- 产品目录、搜索结果和产品详情
- 开放式联盟广告组
- 随机 CTA 转化池
- R2 图片上传、缩略图和删除队列
- D1 数据库迁移
- 管理员登录与会话鉴权
- Robots、Sitemap、Canonical、Open Graph 和 JSON-LD
- GitHub Actions 自动验证、部署、线上 Smoke Test 和 Worker 回滚

项目不使用 React，也不是大型 SPA。公共页面由 Astro SSR 输出 HTML，交互使用少量原生 JavaScript。

## 通用版本定位

这是一个可重复部署的通用商业目录模板，不针对某一个具体行业写死结构或视觉。

### 内容结构

```text
站点
└── 1–4 个分区
    ├── 可选分类
    ├── 产品目录
    ├── 产品详情
    ├── 联盟广告组
    └── CTA 转化组
```

分区是否存在有效分类，决定分区首页的内容模式：

- 有有效分类：分区首页展示分类入口，分类页展示产品目录。
- 没有有效分类：分区首页本身就是产品目录页。
- 产品详情页始终存在，并且只服务当前产品转化。

有效分类建议定义为：至少存在一个已发布分类，并且该分类下存在已发布产品。

### 公共访问链路

```text
进入默认分区
  ↓
浏览分区首页、分类目录或搜索结果
  ↓
查看产品图片和标题
  ↓
进入产品详情
  ↓
阅读正文并点击 CTA
  ↓
转化池实时选择可用链接或短信资源
  ↓
打开或复制目标内容
```

## 视觉主题

公共站点使用面向美国市场的中性通用电商视觉，参考主流电商目录的清晰层级、图片优先和可预期交互，但不复制任何平台品牌。

### 视觉原则

- 白色主背景，辅助背景使用浅灰色。
- 深灰正文、灰色边框和单一克制强调色。
- 不使用渐变、霓虹、玻璃拟态、粉色主题或厚重阴影。
- 使用系统字体，不下载外部字体。
- 产品图片是主要视觉内容，文字信息保持简洁。
- 产品卡片为“图片在上、标题在下”，标题不覆盖图片。
- Header 保留菜单或返回、居中 Logo、行内展开搜索。
- Footer 只保留站点说明、分区入口、Privacy、Disclaimer 和版权。
- Hero/广告区域本身不额外叠加系统标题或 CTA；广告文案由广告素材负责。
- 不增加精选产品、最新产品、推荐模块、价格、库存、SKU、购物车、评分、徽章、筛选侧栏或排序系统。

### 响应式层级

| 设备 | 断点 | 产品列数 | 页面特征 |
|---|---:|---:|---|
| 手机 | `<768px` | 2 列 | 紧凑 Header、固定分区导航、固定详情 CTA |
| 平板 | `768–1099px` | 3 列 | 独立平板宽度、单列产品详情构图 |
| PC | `≥1100px` | 4 列 | 宽屏容器、真正的左右产品详情布局 |
| 宽屏 PC | `≥1400px` | 5 列 | 更宽目录画布，不按平板布局放大 |

底部分区切换支持 1–4 个分区。按钮宽度按数量自动均分，不写死单个按钮宽度。

样式入口统一为：

```text
src/styles/public-system.css
src/styles/admin-system.css
```

Layout 不直接导入内部样式层。`scripts/verify-style-entrypoints.mjs` 在 CI 中锁定样式加载顺序，避免继续叠加无序覆盖。

## 页面广告范围

广告允许出现在：

- 分区首页
- 产品目录页
- 产品搜索结果页

广告禁止出现在：

- 产品详情页
- Privacy
- Disclaimer
- 后台页面
- 404 和错误页

产品详情页是最终转化页，必须做到：

- 不查询广告数据
- 不创建广告容器
- 不加载外部图片或 GIF
- 不执行联盟代码
- 不触发横幅、竖版或弹窗广告

## 联盟广告系统要求

### 核心模型

广告系统不预设固定的“移动端广告组”“PC 广告组”，也不限制广告组数量。后台提供开放式广告组管理：

```text
分区
└── 任意数量广告组
    └── 任意数量广告
```

广告组用于归类和控制一批广告。设备属性保存在广告组上，避免每条广告重复写入设备范围，也便于 D1 按分区和设备建立索引查询。

### 新增广告组

新增广告组窗口包含：

```text
广告组名称
设备类型：移动端 / PC端
状态：启用 / 停用
```

规则：

- 广告组名称自由填写。
- 同一分区可以创建多个移动端广告组和多个 PC 广告组。
- 广告组数量不设业务上限。
- 新增时状态默认选择“启用”。
- 可以随时编辑名称、设备和状态。
- 可以删除广告组；删除时必须提示该组内广告会一并删除。
- 停用广告组后，组内所有广告都不参与前台候选池。
- 平板暂时归入移动端设备类型。

### 新增广告

进入广告组后点击“新增广告”，弹出广告信息录入窗口。

公共字段：

```text
广告名称
展示类型：横幅 / 竖版 / 弹窗
广告类型：上传图片 / 外部图片或 GIF / 联盟代码
状态：启用 / 停用
```

规则：

- 广告继承所属广告组的设备类型，不再单独选择设备。
- 一个广告组中的广告数量不设业务上限。
- 新增广告时状态默认选择“启用”。
- 广告可以随时编辑、停用和删除。
- 编辑窗口必须回显真实状态，不能每次打开都重置为启用。
- 切换广告类型时，必须清理旧类型不再使用的字段，避免残留数据。

### 展示类型

每条广告独立选择：

```text
banner   横幅广告
vertical 竖版广告
modal    弹窗广告
```

广告组不限定展示类型。同一个移动端或 PC 广告组可以混合横幅、竖版和弹窗广告。

#### 横幅广告

- 分区首页：显示在主要内容前或现有 Hero 区域。
- 产品目录和搜索结果：作为独占整行的内容广告插入产品列表。
- 移动端产品目录一行 2 个产品，每 3 行即每 6 个产品形成一个横幅插入点。
- 横幅不伪装成产品卡片，必须显示 `Ad` 或 `Sponsored`。
- 同一个页面存在多个插入点时，优先使用不同广告；候选不足时才允许重复。

#### 竖版广告

- PC：优先显示在主内容外侧的右侧广告轨道，可使用 sticky，但必须在 Footer 前停止。
- PC 广告加载成功后不能压缩原有主内容宽度。
- 移动端和平板：作为独立内容模块居中插入，不强行裁切成横幅。
- 屏幕空间不足时不展示，不能拉伸或破坏素材比例。

#### 弹窗广告

- 只允许在分区首页、产品目录页和搜索结果页触发。
- 页面主要内容完成后才允许加载。
- 用户产生滚动或点击后再允许出现。
- 素材加载成功后才创建并显示弹窗。
- 关闭按钮立即可用，不设置关闭倒计时。
- 不自动跳转，不自动打开新窗口，不播放声音。
- 同一个页面访问最多出现一次。
- 用户关闭后，本次会话不再显示。
- 产品详情页永远不触发弹窗。

### 广告类型

#### 1. 上传图片

支持：

```text
JPG
PNG
WebP
```

字段：

```text
上传图片
目标链接
打开方式：当前窗口 / 新窗口
```

要求：

- 图片宽高由系统自动读取。
- 不支持上传 GIF。
- 上传图片保存到 R2，并在 D1 保存图片资产引用。
- 目标链接必须是有效的 HTTP 或 HTTPS 地址。

#### 2. 外部图片或 GIF

支持远程：

```text
JPG
PNG
WebP
GIF
```

字段：

```text
素材 URL
目标链接
声明宽度
声明高度
打开方式
```

要求：

- 素材 URL 和点击目标 URL 必须分开保存。
- GIF 只使用外部 URL，不上传到 R2。
- 外部素材在后台需要提供加载预览。
- 加载失败的素材不能留下破图或空白区域。
- 例如：`https://www.imglnkx.com/8780/DESIGN-19806_Neon_300100_6.gif` 属于外部 GIF 素材 URL，仍需另外填写联盟点击链接。

#### 3. 联盟代码

支持：

- HTML
- iframe
- JavaScript
- `<a><img></a>`
- 图片与追踪像素组合代码
- 联盟平台提供的完整嵌入代码

字段：

```text
联盟代码
声明宽度
声明高度
可选目标链接
```

要求：

- 联盟代码通过隔离 iframe 加载，不能直接写入主页面 DOM。
- 代码内部已有跳转链接时，额外目标链接可以为空。
- 后台预览时才执行代码，广告列表页不能直接运行全部第三方脚本。

### 广告组和页面关系

广告组不绑定具体页面，也不绑定固定广告位。

只要满足以下条件，广告就进入当前页面候选池：

```text
所属分区 = 当前分区
广告组设备 = 当前设备
广告组状态 = enabled
广告状态 = enabled
页面属于允许广告的公开页面
```

同一分区、同一设备下的所有启用广告组合并为候选池。

### 随机抽取和 D1/Workers 用量

预计每个分区总广告量约 20 条：移动端约 10 条，PC 端约 10 条。该规模不需要复杂权重、排序轮播或独立缓存系统。

要求：

- 不使用广告权重。
- 不使用顺序轮播。
- 不使用定时自动切换。
- 不使用 `ORDER BY RANDOM()`。
- 不使用 `array.sort(() => Math.random() - 0.5)`。
- 页面只发起一次广告候选请求。
- D1 使用索引查询当前分区和设备下的候选广告。
- Worker 使用 `Math.random()` 直接随机下标选择。
- 一个页面需要多个列表广告时，一次查询候选集合，在 Worker 中按展示点分配；同页优先不重复。
- 抽中的素材加载失败时，从剩余候选中重新尝试；全部失败则不显示。

推荐查询路径：

```text
当前分区
  ↓
当前设备
  ↓
启用广告组
  ↓
启用广告
  ↓
Worker 随机选择
```

推荐索引：

```sql
CREATE INDEX idx_ad_pools_channel_device_status
ON ad_pools(channel_id, device_type, status);

CREATE INDEX idx_advertisements_pool_status_type
ON advertisements(pool_id, status, display_type);
```

### 广告加载行为

广告初始状态必须是：

```text
没有广告 DOM
没有占位框
没有空白区域
没有骨架屏
不加载远程图片或 GIF
不执行联盟代码
```

加载流程：

```text
页面主要内容完成
  ↓
请求候选广告
  ↓
隐藏加载选中素材
  ↓
确认素材加载成功
  ↓
创建广告区域
  ↓
轻微淡入显示
```

失败时：

- 不显示前台错误。
- 不留下空白。
- 不改变页面原有布局。
- PC 侧栏广告不能在加载后压缩产品网格。

图片和 GIF 链接使用：

```html
rel="sponsored noopener noreferrer"
```

所有广告外层显示 `Ad` 或 `Sponsored`。

## 推荐素材尺寸

尺寸是运营建议，不作为唯一允许值。系统应读取实际尺寸或使用声明尺寸，并按原始比例显示，不强制裁切成另一种广告形态。

### 广告素材

| 设备与形式 | 首选尺寸 | 可接受尺寸 | 说明 |
|---|---:|---:|---|
| 移动端横幅 | 300×100 | 320×100、320×50 | 适合分区首页和列表整行广告 |
| 移动端竖版 | 300×600 | 300×250 | 作为内容模块居中展示 |
| 移动端弹窗 | 300×250 | 最大约 320×480 | 不能超出视口，保留关闭空间 |
| PC 横幅 | 970×250 | 970×90、728×90 | 作为宽内容广告 |
| PC 竖版 | 300×600 | 160×600、300×250 | PC 右侧广告轨道首选 |
| PC 弹窗 | 600×400 | 300×250、300×600 | 必须适配当前视口 |

素材制作要求：

- 文字、Logo 和 CTA 距离边缘至少保留约 10% 安全区。
- 不依靠 CSS 将横幅裁成竖版或将竖版拉成横幅。
- GIF 建议控制在 1.5 MB 以内，避免首屏和滚动加载明显变慢。
- GIF 不使用高频闪烁或自动声音。
- 联盟代码必须声明宽度和高度，便于前端选择合适容器。

### 产品图片

产品目录采用 4:5 竖向图片，推荐统一素材比例：

| 用途 | 推荐尺寸 | 最低建议 | 输出方式 |
|---|---:|---:|---|
| 产品封面原图 | 1200×1500 | 800×1000 | JPG、PNG 或 WebP |
| 产品详情图库 | 1200×1500 | 800×1000 | 同一产品尽量统一比例 |
| 目录缩略图 | 系统生成最长边约 480px | — | 独立 WebP 缩略图 |
| Logo | 建议横向 600×180 | 300×90 | 透明 PNG 或 WebP |
| Favicon | 512×512 | 192×192 | 正方形 PNG 或 WebP |

产品图要求：

- 主要主体居中，边缘保留安全区。
- 同一产品图库尽量使用相同宽高比，避免详情页切换时跳动。
- 不在目录中读取完整原图。
- 产品封面必须具备目录缩略图，数据库约束拒绝不完整媒体关系。
- 产品图片不叠加系统价格、评分、库存或促销徽章。

## 建议数据结构

广告组：

```text
ad_pools

id
channel_id
name
device_type       mobile / desktop
status            enabled / disabled，默认 enabled
created_at
updated_at
```

广告：

```text
advertisements

id
pool_id
name

display_type      banner / vertical / modal
creative_type     uploaded_image / external_media / embed_code

image_asset_id
media_url
embed_code
target_url

declared_width
declared_height
open_mode         same / new
status            enabled / disabled，默认 enabled

created_at
updated_at
```

字段约束：

- `uploaded_image`：`image_asset_id` 和 `target_url` 必填。
- `external_media`：`media_url`、`target_url`、声明宽高必填。
- `embed_code`：`embed_code` 和声明宽高必填，`target_url` 可为空。
- 一条广告只能使用一种素材来源。
- 删除广告组时，组内广告级联删除。

## 部署架构

```text
GitHub Repository
  ↓
GitHub Actions
  ├── 安装依赖
  ├── 生成 Cloudflare 类型
  ├── Astro / TypeScript 检查
  ├── Node 自动化测试
  ├── 本地 D1 Migration 验证
  ├── 数据完整性验证
  ├── Astro SSR 构建
  ├── Headless Chrome 浏览器检查
  ├── 记录 D1 Time Travel 书签
  ├── 应用远程 D1 Migration
  ├── Wrangler Deploy
  ├── 生产 Smoke Test
  └── 失败时自动回滚 Worker
        ↓
Cloudflare Worker
  ├── Astro SSR
  ├── D1 Binding
  ├── R2 Binding
  └── Static Assets
```

一个仓库对应一个 Worker、一个 D1 数据库和一个 R2 Bucket。前台、后台及 API 均由同一个 Worker 提供。

实现约束：

- 不使用 React
- 不使用 ORM
- 不引入 Hono
- D1 使用 Prepared Statements 与 SQL Migration
- 图片由浏览器直接从 R2 公共域名加载
- 项目自身不实现访问、广告点击、广告曝光或收益统计
- 按配置加载 GA4 和 Meta Pixel
- 不增加广告权重、预算、地理定向、用户画像或联盟平台 API 对接

## 主要路由

```text
/                                      跳转到默认分区，未设置时进入第一个已发布分区
/{channel}                             分区首页；无有效分类时同时是产品目录
/{channel}?category={category}         未配置分类筛选组时的产品筛选
/{channel}/category/{category}         配置分类筛选组时的分类产品目录
/{channel}/product/{product}           产品详情，永远不展示广告
/{channel}/search?q=...                搜索结果，可展示广告
/privacy                               隐私政策
/disclaimer                            免责声明
/robots.txt                            Robots
/sitemap.xml                           动态 Sitemap
/admin                                 后台入口
/admin/login                           后台登录
/api/admin/*                           后台接口
/api/public/*                          公共接口
/api/health                            健康检查
/go/*                                  CTA 转化解析
```

## SEO

公共页面支持：

- Canonical URL
- Robots Meta
- Open Graph
- Twitter Card
- 动态 Sitemap
- 产品 `Product` JSON-LD
- 产品和分类 `BreadcrumbList`
- 分类目录 `ItemList`
- 正文自动生成产品 Meta Description

当 `/` 配置为默认分区跳转时，Sitemap 不提交该跳转 URL。

广告链接必须明确标记为赞助链接，页面 Privacy 或 Disclaimer 应包含联盟佣金关系说明。

## 搜索

搜索覆盖：

- 分类名称
- 产品标题
- 产品标签

输入限制为 48 UTF-8 字节。系统移除 SQL LIKE 通配字符，并在结果中优先排列标题精确匹配、标题前缀匹配和标题包含匹配，标签匹配作为补充。

搜索结果允许展示广告，但产品详情页不展示广告。

## 生产部署验证

生产部署完成后，CI 自动检查：

```text
/api/health
/robots.txt
/
/{channel}/category/{category}
/{channel}/product/{product}
```

部署前的本地 Worker 与 Headless Chrome Smoke Test 还会验证搜索结果、公开产品分页 API、CTA 转化解析接口，以及手机、平板和 PC 三种视口的真实渲染。

广告模块实施后，Smoke Test 还必须验证：

- 允许广告的页面可以延迟加载广告。
- 产品详情页不请求或展示任何广告。
- 外部素材失败不留下空白。
- 横幅、竖版和弹窗使用正确容器。
- 手机、平板和 PC 不发生广告遮挡 CTA、导航或 Footer。

如果 Worker 已部署但 Smoke Test 失败，工作流执行 Worker 自动回滚。D1 不自动恢复，因为 Time Travel Restore 会覆盖生产数据库并取消进行中的查询。

失败诊断附件包含：

- 部署前 Worker 状态
- 迁移前 D1 Time Travel 书签
- 构建和部署日志
- 浏览器 DOM 与手机、平板、PC 截图
- Worker 回滚日志
- `recovery.md` 恢复指南

可通过可选 GitHub Secret `PRODUCTION_ORIGIN` 指定正式域名。未配置时，Smoke Test 从 Wrangler 部署输出提取 `workers.dev` 地址。

## 数据更新规则

后台保存以下内容后不需要重新部署：

- 站点设置
- 分区
- 分类筛选
- 分类
- 产品与图库
- 广告组和广告素材
- CTA 转化资源
- Logo、Favicon 和 R2 公共域名
- GA4 和 Meta Pixel ID

只有代码、依赖、数据库结构或部署配置发生变化时才需要重新部署。

## 性能与 Cloudflare 免费额度原则

- 首次目录读取 20 条产品。
- Load More 每次加载 20 条。
- 目录查询不读取正文和完整图库。
- 目录只读取约 480px 独立 WebP 缩略图，详情读取主图。
- 图片直接从 R2 自定义域名加载。
- 视口外图片懒加载。
- 公共页面不加载大型客户端框架。
- 广告候选数据每页只请求一次。
- 广告查询使用分区、设备、状态和展示类型索引。
- 约 10 条移动端广告和 10 条 PC 广告直接由 Worker 随机下标选择，不进行完整排序。
- 广告素材在真正需要展示时才加载。
- 页面初始不预留广告空白，不创建不可见的大尺寸容器。
- 外部 GIF 和联盟脚本不阻塞首屏主要内容。
- 生产日志采样目标为 1%，避免不必要的 Workers 日志额度消耗。

## 维护原则

- 功能行为与视觉系统分离。
- 模板只保留一个干净的 `0001_initial.sql`，新站从空 D1 数据库安装。
- 测试数据和旧图片不进入模板，不保留旧字段兼容或原图回退。
- 产品图片必须同时具备详情主图和目录缩略图，数据库约束拒绝不完整关系。
- 广告系统不能继续依赖 `channel.hero_ad_pool_id` 的单一 Hero 绑定模型。
- 广告数据结构变更必须通过 D1 Migration 和完整性验证。
- 新增广告组和广告默认启用，但必须保留停用能力。
- 广告页面范围写死在公共页面逻辑中，不在后台增加复杂页面绑定。
- 产品详情页保持纯转化，不增加任何广告。
- 部署失败先查看 CI 诊断附件和 `recovery.md`。
- 不在源码或配置文件中提交生产 Secrets。
